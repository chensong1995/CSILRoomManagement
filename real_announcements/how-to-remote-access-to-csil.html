Remote Access (Remote Login) to CSIL FAQ

Overview

We offer numerous ways for students to access CSIL resources from almost anywhere. For course specific servers, please follow your course documentation, or contact the course instructor.

Remote Access (Remote Login) to CSIL Linux CPU Servers

We provide several CSIL Linux CPU Servers for student's use from off-campus - please follow the link for the names and other important information.

You can use ssh (secure-shell) to connect to these servers with a text-only interface. You may have to install a free ssh Windows client on your Windows machine. Search Google for an ssh client - we recommend PuTTY for Windows users. Apple OS comes with ssh.

After you have installed your ssh client and you've started it up, connect to a CSIL Linux CPU server on port 24, e.g.

ssh -p 24 <name of the cpu server>

Just a reminder: please logoff from the server after finishing your work.

Remote Access (Remote Login) to Linux workstations

First, consider using a CSIL CPU Server, as described above. They are always booted in Linux and are meant for shared use.

* WARNING * There is no guarantee the access to CSIL workstation remotely since all of our workstations dualboot.

You can use ssh (secure-shell) to connect to any Unix/Linux computer with a text-only interface. (You may have to install a free ssh Windows client on your Windows machine. Search Google for an ssh client - we recommend PuTTY for Windows users).

After you have installed your ssh client and you've started it up, connect to a CSIL Unix/Linux system on port 24.  As the default is port 22 you will have to specify port 24 somewhere, depending on your ssh client, e.g.

ssh -p 24 linux_machine.csil.sfu.ca
ssh -p 24 linux_machine.cs.surrey.sfu.ca

(see the CSIL Burnaby layout to find a CSIL Burnaby machine to connect to, and the CSIL Surrey layout to find a CSIL Surrey machine).

Please note:

.) All of CSIL workstations dualboot Ubuntu with Windows. When a computer is booted to Windows, it won't accept SSH connections; please try another one, you may want to use CSIL Linux CPU Server .

.) If you like to have GUI interface to a CSIL Linux workstation, please check out this Q&A.

Access SFU Home

Remember: all your course work should be saved onto your SFU Home.

Launch an ssh client tool (such as PuTTY) from your home computer, connecting to

fraser.sfu.ca

with your campus userid and password. At the prompt on fraser, you can use unix commands such as ls to work with your files. PuTTY includes an scp (secure copy) tool for file transfer. See the PuTTY documentation.

Access CSIL Subversion repository

A good choice for transferring your work between CSIL and other computers is to use your subversion repository on punch.csil.sfu.ca - one is created for each CMPT course in which you are registered. Please check out our subversion FAQ page for more instructions.
